# 🤖 PhpTravels Automation Testing Project

## 📌 Project Overview
This project demonstrates automated testing of a real-world travel booking web application (PhpTravels). 
The automation framework is designed to validate core functionalities such as user authentication, hotel search, and booking processes using Selenium.

## 🎯 Objectives
- Automate repetitive test scenarios for efficiency
- Ensure application stability through regression testing
- Validate end-to-end user workflows
- Improve test coverage and reduce manual effort

## 🛠️ Tools & Technologies
- Selenium WebDriver
- Java
- TestNG
- Maven
- Page Object Model (POM)
- Git & GitHub

## 📋 Automation Scope
- Functional Testing
- Regression Testing
- UI Automation Testing

## 🔄 Key Test Scenarios
- User Login & Logout  
- Hotel Search Functionality  
- Booking Process Validation  
- Form Validation  
- Navigation Testing  

## 🧩 Framework Design
- Page Object Model (POM) for better maintainability  
- Reusable methods and utilities  
- TestNG for test execution and reporting  
- Maven for dependency management  

## 🧪 Features Automated
- End-to-end user journey automation   
- Assertion-based validation  
- Error handling and synchronization  

## 🐞 Defect Handling
- Identified UI and functional issues during automation  
- Logged defects with proper steps and screenshots  
- Retested after bug fixes  

## 🔄 Automation Workflow
1. Requirement Analysis  
2. Test Case Identification  
3. Framework Setup  
4. Script Development  
5. Test Execution  
6. Reporting & Debugging  

## 📊 Outcome
- Reduced manual testing effort  
- Faster regression testing cycle  
- Improved software quality and reliability
